# Отчёт о выпуске 0.7.0

Дата: 2026-07-17.

`0.7.0` объединил ранее раздельные map/campaign/route, economy/combat, loading/save и hyperspace
наработки в одну проверяемую поставку. Это внутренний этап до появления имени
StarsectorPrepatcher, но его результаты важны как baseline для последующих исправлений.

## Что было добавлено

- линейная reconciliation замена горячего `retainAll` на карте;
- lifecycle-bound кэши campaign, location, Intel и Economy;
- reusable snapshots и scratch collections с защитой reentrancy и exceptional exit;
- spatial indexes для labels, routes и comm-relay candidates;
- streaming loading/save paths, literal rules parser и progress redraw throttle;
- отдельный hyperspace javaagent для terrain culling, Y clamp, no-op layer, automaton buffers и
  moving-starfield cleanup.

Каждая группа получила отдельный `patch.*` switch. Main agent применяет независимый structural и
data-flow matching, hyperspace agent — точные per-class guards. Неизвестный или частично изменённый
target остаётся vanilla.

## Исправления интеграции

Исходная hyperspace-ветка не содержала рабочего compiled agent. При объединении были исправлены:

- неверный enum import;
- runtime export JDK-internal ASM packages без дополнительных vmparams flags;
- target registry с per-class hashes вместо привязки к целому JAR;
- Y-clamp matcher и выбор фактического `yEnd`, а не горизонтального `xEnd`;
- полное покрытие target-классов и ненулевой exit code при partial target set;
- unsafe fuzzy patches, которые были удалены как недоказанные.

## Проверка

Structural suite преобразовал 23 класса и проверил 1 630 методов. Отдельно прошли:

- negative и idempotency cases для missing/ambiguous/foreign/partial sites;
- lifecycle/GC со сменой campaign generation и сборкой старого engine через `WeakReference`;
- snapshot isolation, reuse, reentrancy и cleanup на exceptional exit;
- Economy fingerprint, comm-relay ordering/fallback и bounded failed-build retry;
- UTF-8 boundary/close, literal-parser differential и progress-frequency tests;
- hyperspace offline verification: 4 targets patched, 123 methods verified, failed=0;
- combined startup smoke двух javaagent на Microsoft OpenJDK 17.

## Производительность

Первый большой modded smoke подтвердил активность оптимизированных путей: `retainAll` выполнился
13 383 раза, label index сократил 1 019 091 исходных элементов до 2 798 candidates, Intel callback
cache дал 8 355/985 hits/misses, route jump index — 6 636 hits при одной сборке без fallback.

Same-save allocation-прогон снизил steady allocation приблизительно с `4.347` до
`2.184 MiB/frame`. Одновременно он обнаружил CPU-регрессию: очистка пустой `IdentityHashMap`
увеличила steady frame time примерно с `11.09` до `29.67 ms`. Это стало входом для исправлений
`0.7.1`; поэтому allocation-выигрыш `0.7.0` нельзя трактовать как итоговый FPS-результат.

## Остаточные риски

ASM verification доказывает корректность bytecode/control flow, но не визуальную эквивалентность и
не величину ускорения. Terrain culling, Y clamp и no-op layer требуют графического A/B на реальном
наборе модов и сохраняют независимые kill switches.

Сырые воспроизводимые результаты создаются в `.build/reports/` и не включаются в дистрибутив из-за
timestamps и локальных путей.
