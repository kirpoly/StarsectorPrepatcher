# Отчёт о выпуске 0.8.0

Дата: 2026-07-17.

`0.8.0` — первый выпуск под именем **StarsectorPrepatcher**. Имя отражает архитектуру: startup
javaagent регистрирует transformer до обычной загрузки game/mod classes и применяет guarded patches
при первом определении целевого класса.

## Что изменено

- mod id: `starsector_prepatcher`;
- каталог: `StarsectorPrepatcher`;
- main agent: `agent/StarsectorPrepatcherAgent.jar`;
- hyperspace agent: `agent/StarsectorPrepatcherHyperspaceAgent.jar`;
- bootstrap: `jars/StarsectorPrepatcherBootstrap.jar`;
- конфигурация: `prepatcher.properties` и `hyperspace-prepatcher.properties`;
- логи: `logs/prepatcher.log` и `logs/prepatcher-hyperspace.log`;
- Java namespace: `com.starsector.prepatcher`;
- system properties: `starsector.prepatcher.*`.

Переходная совместимость с внутренними именами до первого публичного выпуска намеренно не
предоставляется. Installer управляет только двумя javaagent StarsectorPrepatcher, а runtime читает
только канонические конфигурационные файлы.

## Направление API

Проект планирует предоставить документированный API к возможностям, отсутствующим в публичном API
игры. Планируемый namespace — `com.starsector.prepatcher.api`. В `0.8.0` API ещё не опубликован:
внутренние hooks не являются совместимой поверхностью и могут меняться до появления спецификации и
отдельных compatibility tests.

## Функциональное влияние

Переименование не урезает и не расширяет оптимизационные патчи `0.7.1`, поэтому самостоятельного
изменения FPS от `0.8.0` не ожидается. Performance baseline, причины hot-path исправлений и
ожидаемые telemetry-сигналы описаны в [`0.7.1.md`](0.7.1.md).

Prepatcher не добавляет сериализуемые campaign objects: временные кэши и scratch buffers существуют
только в памяти текущей JVM.

## Проверка

Release gate проверяет mod identity, package namespace, artifact names, manifest versions,
language switch в обоих README, полный checksum manifest и отсутствие прежних runtime paths.
Дополнительно проходят structural/negative/idempotency, lifecycle/GC, runtime, hyperspace и
combined startup suites.

Сырые результаты остаются в `.build/reports/`; подтверждённые пользовательские изменения — в
[`CHANGELOG.md`](../../CHANGELOG.md), а причины и измеренное влияние патчей — в этих release notes.
