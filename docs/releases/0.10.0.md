# Отчёт о выпуске 0.10.0

Дата: 2026-07-21.

## Назначение

`0.10.0` объединяет unified market scheduler, structural fast-forward presentation patches,
локальные composition contracts для пересекающихся bytecode surfaces и CoreScript core-worlds
optimizer. Основная цель выпуска — сократить frame-time и retained-memory overhead больших кампаний,
не ослабляя fail-open поведение и не создавая второй javaagent.

Это первый поставляемый профильный контракт, где `prepatcher.properties` полностью совпадает с
`profiles/aggressive.properties`. `profiles/debug.properties` наследует тот же aggressive baseline и
добавляет только ограниченный набор observers, CSV/stack sampling, verbose logging и runtime metrics.
Оба отношения проверяются автоматически.

## Market scheduler и economy

- Один scheduler обслуживает Economy-loop, planet-condition и синхронизированные core call sites.
- Deferred debt привязан к weak identity keys, сбрасывается перед save и немедленно синхронизируется
  при прямом модовом `MarketAPI.advance(float)`.
- Construction policy использует raw `BaseIndustry.building`; upgrade-only и virtual/raw
  расхождения остаются диагностикой и не удерживают рынок в full-rate.
- Economy snapshots, location cache и scheduler source объединены в один атомарный patch plan.
- Market snapshots, commodity temporal fast path и entry observation также применяются одной
  транзакцией с общей feature mask и combined postcondition.

Runtime regressions покрывают capability/lifecycle gate, batch cadence, callback failures,
pre-save abort/restore, direct synchronization, weak-key GC, telemetry reset и повторный запуск
stats worker. Observer paths не удерживают сильные ссылки на market или industry objects.

## Presentation и композиция патчей

Fast-forward presentation transformer больше не зависит от whole-JAR SHA-256. Двадцать четыре
target-класса используют локальные structural plans, owner marker и global feature mask. Для пяти
классов, затрагиваемых также structural transformer-ом, задан явный порядок
presentation → structural и обязательная повторная проверка presentation postcondition после каждого
structural commit.

Default/aggressive включают global animations, sensor faders, slipstream particles и particle
emitters. Safe сохраняет более узкий presentation surface. Режим visual time остаётся `realtime`,
поэтому визуальные callback-и получают один обычный amount на outer frame.

## Core-worlds optimizer

Terminal `Misc.computeCoreWorldsExtent()` внутри `CoreScript.advance(float)` заменяется typed helper-ом
общего runtime payload. Matcher привязан к class/method descriptor, terminal semantic call site и
локальному data flow `Global.getSector()` вместо номера local slot или точного hash класса/JAR.

Проверки покрывают vanilla application, post-state, idempotent reprocessing, malformed/ambiguous
surfaces, actual-agent execution и FR-like split-loader topology.

## Диагностические профили

Из default, safe и aggressive удалены:

- semantic-risk и construction observers;
- direct-market CSV и stack sampling;
- verbose presentation logging и presentation metrics;
- периодический stats worker.

Они включены только в debug-профиле. Consistency test требует одновременно:

1. точного равенства всех активных свойств default и aggressive;
2. наследования debug всех aggressive-значений;
3. ровно тринадцати утверждённых дополнительных debug-ключей;
4. безопасных code defaults для отсутствующих диагностических параметров.

## Проверка выпуска

Перед сборкой архива выполнены:

- documentation/version/checksum consistency;
- full enabled patch order и idempotent structural reprocessing;
- combined postconditions пересекающихся patch surfaces;
- presentation structural/runtime/actual-agent suites;
- market scheduler, direct observer, economy, commodity, temp-mod и cache-maintenance regressions;
- hyperspace structural/runtime regressions;
- startup smoke;
- Faster Rendering split-loader smoke и runtime payload inventory.

## Остаточные риски

- Aggressive presentation groups намеренно имеют более широкую семантическую поверхность, чем safe;
  при несовместимости конкретного мода следует временно использовать `profiles/safe.properties`.
- Debug observers создают sampling/file-I/O overhead и предназначены только для ограниченной сессии.
- Локальный structural contract доказывает совместимость transformation surface, но не заменяет
  длительный campaign differential run для глобального RNG/intercomponent ordering.
- `patch.loadingTextReader` и `patch.startupLogAggregation` остаются отключёнными во всех профилях до
  отдельного успешного mission-startup validation.
